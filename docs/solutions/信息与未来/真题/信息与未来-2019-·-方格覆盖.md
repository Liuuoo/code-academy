# [B3755 [信息与未来 2019] 方格覆盖 - 洛谷](https://www.luogu.com.cn/problem/B3755)

## 题目描述

给定一个 $n\times n$ 的矩形，其中从左上角开始，对角线上连续的 $k$ 个格子中有障碍物。你可以把若干 $1\times2$ 的小矩形放置到该大矩形中，要求是放置的两个小矩形不能占据相同的格子，且不能碰到障碍物。例如下图是 $n=4,k=2$ 的例子，我们放置了 $6$ 个 $1\times2$ 的小矩形。

![](https://cdn.luogu.com.cn/upload/image_hosting/ifmknyb8.png)

给定 $n,k$，请你输出一个方案，使得放置的 $1\times2$ 小矩形尽可能多。可以证明，$n=4,k=2$ 时，至多只能放置 $6$ 个小矩形。
## 输出格式

输出 $n$ 行，每行 $n$ 个整数（用任意数量的空格分隔）。放置的小矩形分别用 $1,2,\cdots$ 编号。不放置小矩形的格子输出 $0$。如有多种最优方案（放置最多数量的小矩形），输出任意一种即可。

## 题解
我们先不去看为什么给定的样例为什么是这样一个放置方式（往往构造题题目给定的构造都不是正确的构造，注意往往）。

先理清约束与目标：
- 约束1：正方形格子
- 约束2：每次覆盖是连在一起的两个
- 约束3：不能放置的格子从坐上到右下且连续
- 目标：构造一种策略，使得覆盖最多

### 思考1：特殊情况
约束很多，我们先尝试一种简单情况：无不能放置的格子，即k=0：

我们可以怎么放置呢？其实很多同学都能想出来一种可行策略：先竖着，再横着或者先横着再竖着。然后验证奇/偶 后发现没问题，那么大致可以确定这是一种合理的方式。

这种情况实际上可以放置`n*n/2`对，实际上以任意一种策略（先横再竖着）都可以放置`n*n/2`对。那为什么这样可行呢？这边引出一个经典模型：棋盘染色模型
#### 棋盘染色模型
如下是一个棋盘，它的特点是，`n*n`，黑白相邻。
> （图片缺失，待补）
如果我们想对一个空白网格染色，并且黑白交替（也就是两两相邻染成一黑一白），会发现一个绝对成立的约束：**任何一个 $1 \times 2$ 的小矩形，必定且只能覆盖一个黑格和一个白格。** **所以能染色的对数为min($count_黑$，$count_白$)**

回到题目，在我们约束中，最强约束就是`n*n`的网格进行覆盖。其次覆盖要求一定是相邻覆盖。很容易发现，其实这就是上述棋盘染色模型的原型。这边我们得到第一个结论：

- **结论1：我们可以将相邻覆盖看作染色棋盘中的黑白相邻染色。**

实际上我们只用考虑奇偶，如果是偶数边长，可以直接严丝合缝地铺满（达到 $n^2 / 2$）即min($count_黑$，$count_白$)；如果是奇数边长，刚好只会剩下一个格子（达到 $(n^2 - 1) / 2$）即min($count_黑$，$count_白$)。

我们可以补充结论一：
-  **结论1：我们可以将相邻覆盖看作染色棋盘中的黑白相邻染色。且最终完全覆盖（成对匹配）数就是黑-白对数。**

于是这种特殊情况就被我们解决了，我们再想一般情况：

我们仍然先分析，这样的去除方法，有什么特点规律。首先他一定是斜的，这直接可以看出，所有不能涂色的位置，颜色都一致。这是我们看出的第二个结论：

### 最终整合

于是现在有两条结论：
-  **结论1：我们可以将相邻覆盖看作染色棋盘中的黑白相邻染色。且最终完全覆盖（成对匹配）数就是黑-白对数。**
- **结论2：所有被ban的位置，对应棋盘颜色都一致。且有规律的从左上到右下。**
实际上k这个约束，使得棋盘中某一种颜色（要么黑要么白）数量减少，那么**最多**的成对匹配数量是多少呢？是 : $$min(count_黑，count_白) $$
于是这边我们看一下，是否能达成最多的数量。

由于棋盘的特殊性，白色旁边一定有黑色，黑色旁边一定有白色，所以（先假定白色数量少）覆盖一个白色一定会覆盖一个黑色，白色数量少，黑色数量多，所以白色一定会被轻松覆盖完。

**原因：** 由于棋盘黑白相间的特殊性，覆盖一个白色一定会消耗一个黑色。因为对角线障碍物的原因，现在白色格子数量少，黑色格子数量多。

*有人可能会担心：会不会出现几个白格子挤在一起，只能抢同一个黑格子的情况？*

*答案是绝对不会。因为对角线的障碍物就像广场中间的一条直线隔离带，它不会制造任何死胡同，两边的空间是完全开阔、连通的。在这种四通八达的空间里，既然整体上黑色（多）> 白色（少），就意味着一旦发生局部争抢，白格子总能通过‘连环退让’的方式，在周围开阔的空间里找到其他的黑格子进行配对。（可以自行验证几个（滑稽））*

所以，既然数量够用，空间又没有死角，数量较少的白色格子一定会被毫无悬念地 100% 覆盖完。最多能放的小矩形数量，就是白色格子的总数。

至此，我们就可以开始尝试实现：
### 解题
- 解法1：
	- 现在这题就变成，寻找较少颜色（x+y的奇偶性即可判断出来），然后遍历该格子的几种方向，当存在某个方向可以覆盖时，进行覆盖（一定可以找到）。
- 解法2：
	- 当你进行解法1的代码设计的时候，其实可以想到，无论较少格子还是较多格子的覆盖，都会把对立颜色格子也覆盖，于是回到了解法一的后面实现，不过这次不需要先找到较少格子。并且会将前面遍历到的位置都覆盖，并且由于我们是从左到右，从上到下，所以左和上一定没了，只有判定右和下是否还有。
代码：
```cpp
#include<bits/stdc++.h>
using namespace std;

const int N=100;
int a[N][N];

int main()
{
    memset(a,-1,sizeof(a)); // 这边注意memset赋值int类型
    int n,k;
    cin>>n>>k;
    for(int i=1;i<=n;i++)
    {
        for(int j=1;j<=n;j++)
        {
            if(i==j && i<=k)
            {
                a[i][j]=0;
            }
        }
    }
    int cnt=1;
    for(int i=1;i<=n;i++)
    {
        for(int j=1;j<=n;j++)
        {
            if(a[i][j]==-1) // 初始
            {
                if(j+1<=n&&a[i][j+1]==-1) // 右边有
                {
                    a[i][j]=cnt;
                    a[i][j+1]=cnt++;
                }
                else if(i+1<=n && a[i+1][j]==-1) // 下面有
                {
                    a[i][j]=cnt;
                    a[i+1][j]=cnt++;
                }
                else a[i][j]=0;
            }
        }
    }
    for(int i=1;i<=n;i++)
    {
        for(int j=1;j<=n;j++)
        {
            cout<<a[i][j]<<" ";
        }
        cout<<'\n';
    }
}
```
这边注意memset赋值int类型，可以顺带讲解一下memset函数赋值原理（区分char,int的不同）
