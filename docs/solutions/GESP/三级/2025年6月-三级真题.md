## 第一部分：单选题精讲 (每题 2 分)

### 第 1 题：原码的取值范围

**题目描述**：8位二进制**原码**能表示的最小整数是？

- **正确答案**：**A (-127)**
    
- **详细解析**：
    
    - **原码 (Sign-Magnitude)**：最高位是符号位（0正1负），其余7位是数值位。
        
    - 最小值构造：符号位为 1 (负)，数值位最大 (1111111)。
        
    - 二进制 `1111 1111` (原码) = $-(2^6 + ... + 2^0) = -127$。
        
    - 注意：8位原码中，`1000 0000` 表示 -0，`0000 0000` 表示 +0。
        
- **【举一反三】重点笔记**：
    
    - **原码范围**：$[-127, +127]$
        
    - **反码范围**：$[-127, +127]$ (同样有两个0)
        
    - **补码范围**：$[-128, +127]$ (只有一个0，多出来的 `1000 0000` 规定为 -128)
        
    - **考试技巧**：看到“最小整数”且提到“8位”，如果是补码选 -128，如果是原码/反码选 -127。
        

### 第 2 题：反码中的零

**题目描述**：反码表示中，零的表示形式有几种？

- **正确答案**：**B (2种)**
    
- **详细解析**：
    
    - 正零：`0000 0000`
        
    - 负零：`1000 0000` (原码) -> `1111 1111` (反码，符号位不变，其余取反)。
        
    - 所以有 +0 和 -0 两种形式。
        
- **【举一反三】**：
    
    - **补码的优势**：补码解决了 +0 和 -0 的二义性，消除了 -0，将其利用为最小值（如 -128），这也就是为什么计算机内部通常使用补码的原因。
        

### 第 3 题：补码转真值

**题目描述**：补码 `1011 1011` 对应的十进制真值是多少？

- **正确答案**：**A (-69)**
    
- **详细解析**：
    
    1. **看符号位**：最高位是 `1`，说明是负数。
        
    2. **求原码**：
        
        - 保留符号位，数值位取反：`1100 0100`
            
        - 末位加 1：`1100 0101` (这里是逆运算：补码转原码 = 取反加1，或者减1取反，结果一样) -> **更正算法**：
            
        - 方法一（减1取反）：`1011 1011` - 1 = `1011 1010` -> 取反(符号位不变) -> `1100 0101`。
            
        - 方法二（取反加1）：`1011 1011` 数值位取反 -> `1100 0100` -> 加1 -> `1100 0101`。
            
    3. **算数值**：`100 0101` (二进制) = $64 + 4 + 1 = 69$。
        
    4. **加符号**：-69。
        
- **【举一反三】**：
    
    - 正数的补码 = 原码本身。
        
    - **速算技巧**：补码 `1011 1011` = $-1 \times 2^7 + 0 \times 2^6 + 1 \times 2^5 + 1 \times 2^4 + 1 \times 2^3 + 0 \times 2^2 + 1 \times 2^1 + 1 \times 2^0$ $= -128 + 32 + 16 + 8 + 2 + 1 = -128 + 59 = -69$。 _(这种“权重法”算补码非常快，最高位权重是负的)_。
        

### 第 4 题：补码除法（右移）

**题目描述**：X的补码 `0000 1010`，求 X/2 的补码。

- **正确答案**：**A**
    
- **详细解析**：
    
    - `0000 1010` 是正数 10。
        
    - 10 / 2 = 5。
        
    - 5 的补码是 `0000 0101`。
        
    - **位运算视角**：整数除以 2 等价于**右移一位**。对于补码，算术右移（保留符号位）。
        
    - `0000 1010` >> 1 = `0000 0101`。
        

### 第 5 题：二进制小数转十进制

**题目描述**：`1101.101` 转十进制。

- **正确答案**：**A (13.625)**
    
- **详细解析**：
    
    - **整数部分**：`1101` = $8+4+1 = 13$。
        
    - **小数部分**：
        
        - 第一位 `.1` = $2^{-1} = 0.5$
            
        - 第二位 `.0` = 0
            
        - 第三位 `.1` = $2^{-3} = 0.125$
            
    - 总和：$13 + 0.5 + 0.125 = 13.625$。
        

### 第 6 题：溢出判断

**题目描述**：补码加法，符号位无进位，次高位有进位，说明？

- **正确答案**：**B (发生上溢/正溢出)**
    
- **详细解析**：
    
    - **溢出判别准则**：`溢出 = 符号位进位 XOR 次高位进位`。
        
    - 题目中：符号位进位($C_f$) = 0，次高位进位($C_{max}$) = 1。
        
    - $0 \oplus 1 = 1$ -> 发生溢出。
        
    - 具体分析：
        
        - 两个正数相加（符号位0+0），若次高位有进位（1），则结果符号位变成1（负），这是**正溢出**（Positive Overflow），即结果太大，正数变成了负数。
            

### 第 7 题：八进制转十进制

**题目描述**：八进制 `35.6` 转十进制。

- **正确答案**：**A (29.75)**
    
- **详细解析**：
    
    - 整数 `35` (8进制) = $3 \times 8 + 5 = 29$。
        
    - 小数 `.6` (8进制) = $6 \times 8^{-1} = 6/8 = 0.75$。
        
    - 结果：29.75。
        

### 第 8 题：位运算

**题目描述**：`1010` (??) `1100` 的结果是？（结合选项推测题目是求 OR 或者 AND）

- **题目分析**：
    
    - `1010` (十进制10)
        
    - `1100` (十进制12)
        
    - 若运算是 AND (`&`)：`1000` (8)。选项A符合。
        
    - 若运算是 OR (`|`)：`1110` (14)。选项B符合。
        
    - _注：参考答案通常给出的可能是 B (1110) 或者是 A。根据常规题目设计，若图片模糊，需看选项分布。假设是 Bitwise OR，则为 1110。假设是 AND，则为 1000。_
        
    - **GESP真题回顾**：通常考察 AND/OR/XOR。如果题目文字丢失，我们重点掌握计算方法。
        
    - **假设题目是 OR**：`1|1=1, 0|1=1, 1|0=1, 0|0=0` -> `1110` (B)。
        
    - **假设题目是 AND**：`1&1=1` -> `1000` (A)。
        

### 第 9 题：交换变量

**题目描述**：哪个位运算可以交换 a, b？

- **正确答案**：**A**
    
- **详细解析**：
    
    - 这是经典的**异或交换算法 (XOR Swap)**。
        
    - `a = a ^ b;`
        
    - `b = a ^ b;` (此时 b 变成了原来的 a)
        
    - `a = a ^ b;` (此时 a 变成了原来的 b)
        
- **【重点笔记】**：此方法无需临时变量，但有一个坑：如果 `a` 和 `b` 指向同一个内存地址（即 `&a == &b`），该代码会将变量置为 0。

### 第 11 题：代码填空（数位分离）

**题目描述**：穷举水仙花数类似的逻辑，求中间位 `b`。

- **正确答案**：**B**
    
- **详细解析**：
    
    - `i` 是三位数 (100-999)。
        
    - `b` 代表十位数字。
        
    - 方法：先除以 10 去掉个位，再对 10 取模得到个位。
        
    - `b = (i / 10) % 10`。
        
    - 例如 123 -> 123/10 = 12 -> 12%10 = 2。正确。
        

### 第 12 题：代码填空（小球落地）

**题目描述**：`height /= 2;` 后 `distance` 怎么加？

- **正确答案**：**B (distance += height)**
    
- **详细解析**：
    
    - 逻辑：这是一个模拟题。每次循环模拟一次“落地反弹”的过程。
        
    - 代码逻辑是先 `height /= 2` (高度减半，变为反弹高度)，然后累加距离。
        
    - 如果题目定义 `distance` 是“反弹的高度之和”或者简化的物理模型，B 是最合理的（累加当前的状态）。
        
    - _注意_：如果是求总路程（落下+弹起），通常是 `dist += height * 2`（除了第一次）。但看代码极简，可能是求“第N次反弹后的某种累积值”。结合选项，`height` 已经是减半后的值，`+= height` 符合累加当前高度。

## 第二部分：判断题精讲 (每题 2 分)

1. **String 比较**：`==` 比较地址？ -> **错 (X)**。C++ `string` 重载了 `==`，比较的是**内容**。`char*` 才是比较地址。
    
2. **Substr**：`substr(1,3)` 从下标1开始取3个？ -> **对 (√)**。
    
3. **位运算替代除法**：`x`是浮点数，`x >> 1` 等价于除2？ -> **错 (X)**。位运算（`>>`, `<<`, `&`, `|`）只能用于**整数**。
    
4. **字面量比较**：`string("hello") == "hello"`？ -> **对 (√)**。左边构造了 string 对象，右边是 C 字符串，string 支持与 C 字符串比较。
    
5. **Sort Set**：`sort` 直接排 `set`？ -> **错 (X)**。`std::set` 本身就是有序的（红黑树），且其迭代器不支持随机访问，不能传给 `std::sort`。
    
6. **判断偶数**：`(x&1)==0`？ -> **对 (√)**。二进制末位为0即为偶数，效率比 `%2` 高。
    
7. **Substr 异常**：长度不足抛异常？ -> **错 (X)**。`substr(pos, len)` 中，如果 `pos` 合法但 `len` 超过剩余长度，它会自动截取到末尾，**不会报错**。只有 `pos` 越界才会抛 `out_of_range`。
    
8. **Pow 精度**：`pow` 返回浮点可能不准？ -> **对 (√)**。`pow(2,3)` 可能算出 `7.999999`，转 int 变成 7。整数幂运算建议手写循环或快速幂。
    
9. **枚举类型**：底层可以是 float？ -> **错 (X)**。Enum 底层必须是整型（int, char 等）。
    
10. **返回值转换**：`double f()` 返回 `int` 会自动转？ -> **对 (√)**。发生隐式类型转换。
    

## 第三部分：编程题精讲 (每题 25 分)

### 3.1 奇偶校验 (Parity Check)

**题目核心**： 输入 N 个数，统计这些数二进制表示中 **"1" 的总个数**。如果总个数是奇数，校验码为 1，否则为 0。

**知识点**：

1. **位运算取末位**：`n & 1` 可以取出二进制最低位。
    
2. **右移去掉末位**：`n >> 1` 可以去掉最低位。
    
3. **循环计数**：不断 `n >>= 1` 直到 `n` 变为 0。
    

**代码逻辑**：

```
// 核心片段
int total_ones = 0;
for(int i=0; i<n; i++) {
    int c;
    scanf("%d", &c);
    // 统计 c 中 1 的个数
    while(c > 0) {
        total_ones += (c & 1); // 如果末位是1，计数+1
        c >>= 1;               // 右移一位
    }
}
printf("%d %d\n", total_ones, total_ones & 1); // 输出总数和奇偶性
```

**扩展技巧**：

- 还有一个更快的算法 `c = c & (c-1)` 可以直接消除二进制中最右边的 1，循环次数等于 1 的个数，效率更高（虽然本题数据小不需要）。
    

### 3.2 分糖果 (Candy Distribution)

**题目核心**： N 个小朋友，每个小朋友有一个最低需求 $a_i$。 要求：第 $i$ 个小朋友的糖果数必须 **严格大于** 第 $i-1$ 个小朋友。 求最少总糖果数。

**算法思路 (贪心/递推)**：

1. **贪心策略**：为了总数最少，每个小朋友只给“刚好满足条件”的数量。
    
2. **条件**：
    
    - 条件 1：$Candy[i] \ge a_i$ (满足自身需求)
        
    - 条件 2：$Candy[i] > Candy[i-1]$ (比前一个多，即至少是 $Candy[i-1] + 1$)
        
3. **综合公式**： $Candy[i] = \max(a_i, Candy[i-1] + 1)$
    

**数据范围陷阱**：

- $a_i$ 可达 $10^9$，N 可达 1000。
    
- 总和可能超过 `int` 范围（$10^9 \times 1000 = 10^{12}$）。
    
- **必须使用 `long long` 存储答案**，否则会溢出导致 0 分。
    

**参考代码**：

```
#include <iostream>
#include <algorithm>
using namespace std;

const int MAXN = 1005;
int a[MAXN]; // 存储最低需求
int candies[MAXN]; // 存储实际分配

int main() {
    int n;
    scanf("%d", &n);
    
    long long sum = 0; // 重点：使用 long long 防止溢出
    int prev_candy = 0; // 记录上一个小朋友分到的糖果，第0个虚拟为0
    
    for(int i=1; i<=n; i++) {
        int limit;
        scanf("%d", &limit);
        
        // 核心递推公式
        // 当前分配 = max(当前最低需求, 上一个分配数量 + 1)
        int current_candy = max(limit, prev_candy + 1);
        
        sum += current_candy;
        prev_candy = current_candy; // 更新前一个的值供下一次使用
    }
    
    printf("%lld\n", sum);
    return 0;
}
```
