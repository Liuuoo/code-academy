# GESP 202509 五级 · 有趣的数字和


## 题目描述

如果一个正整数的二进制表示包含奇数个 1，那么小 A 就会认为这个正整数是有趣的。

例如，7 的二进制表示为 (111)2​，包含 1 的个数为 3 个，所以 7 是有趣的。但是 9=(1001)2​ 包含 2 个 1，所以 9 不是有趣的。

给定正整数 l,r，请你统计满足 l≤n≤r 的有趣的整数 n 之和。

## 输入格式

一行，两个正整数 l,r，表示给定的正整数。

## 输出格式

一行，一个正整数，表示 l,r 之间有趣的整数之和。

## 尝试解题与题目的猜测
### 结论一，奇数1的数个数与偶数1的数的个数比较均匀
拿到题目肯定能看出来与二进制有关，但至于什么关系，实在是不好想。我看到了奇偶，于是猜测是否能比较均匀的分割奇数1的数量（以下简称奇数）和偶数1（以下简称偶数）的数量。实际上结果与预期一样，分配比较均匀。这边还是打表得出结论：
```cpp
#include<bits/stdc++.h>
using namespace std;

bool calc(int x)
{
    int cnt=0;
    while(x)
    {
        cnt+=x%2;
        x/=2;
    }
    return (cnt%2);
}

int main()
{
    set<int> S;
    for(int i=1;i<=1000;i++)
    {
        for(int j=i+1;j<=1000;j++)
        {
            int ans=0;
            int l=i,r=j;
            for(int i=l;i<=r;i++)
            {
                if(calc(i)) ans++;
            }
            S.insert(ans-(r-l+1-ans));
        }
    }
    for(auto X:S) cout<<X<<" ";
}
```
最终跑出来，只有五种可能结果：
```
-2 -1 0 1 2
```
- 这边可以将0归结于等分，样例种两个例子就是等分的。
- 相差一，则可能像1-2这样的例子，是某个进位然后截断照成的。由01 -> 10
- 相差二，则是1-4这样的例子。同时进行了两次进位 由01->10->11->100
- 至于更多的，是不可能的。可以证明，但是打表结果在这边，就不过多解释。

### 处理
想用前缀和处理，仍然是需要得到1-x，其中x在`[1-1e9]`的范围，一定会超时。但我突然想到，我们似乎可以不枚举x，而是暴力的或者说用数学的方式来完成1-任意x的和，可能是因为他跟二进制强相关。

现在我们举出一个二进制表示的数字，比如说：
```
1010
```
有一种简单的情况，当我们遇到形如：
```
1000
```
这样的数时，我们很好判断。因为我们可以很容易的算出三位二进制组成的数中的奇数和即：100-111中的奇数和。
然后我们将之推广到一般情况：1010。我们需要处理的是1000-1010中的奇数和，再处理100-111，再到10-11...
现在问题转移到处理1000-1010的问题。这样的数据我们怎么处理？实际上我们仍然可以将之分解。1000-1010这个例子不是很好解释，我们换个例子。
比如10000-101011。那么其中的奇数和，可以看作是0-1、00-10以及0000-1000的奇数的和。于是又回到了先前的特殊情况。

好的，我们回到特殊情况。我们看一下他到底有多特殊。首先l的r的最高位是1，其余位是0，r的所有位都是1。所以我们现在要弄清楚这样的数据有什么规律。

然后有了如下代码：
```cpp
#include<bits/stdc++.h>
using namespace std;

bool calc(int x)
{
    int cnt=0;
    while(x)
    {
        cnt+=x%2;
        x/=2;
    }
    return (cnt%2);
}

int main()
{
    int l=1;
    for(int i=1;i<=12;i++)
    {
        int r=l<<i;
        cout<<l<<" "<<r<<' ';
        int ans=0;
        for(int j=l;j<=r;j++)
        {
            if(calc(j))
            {
                // cout<<j<<" ";
                ans+=j;
            }
        }
        cout<<(ans==(r)*(r+1)/2)<<"\n\n";
    }
}
```
该代码是从找有多少数->判定分布规律->尝试求和发现规律得到的。总之结果就是：

很有规律，很神奇。也就是说，可以使用O(1)得到任意1-1000的值，并且由于10000一定是奇数，也可以很容易得到1-1111的值。于是得到这题的解。
